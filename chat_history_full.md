# Полная история чата

## Сообщение 1 — Пользователь
Мне нужно разработать чат-бота для мессенджера Telegram на языке Python, который будет помогать пользователю в изучении китайского языка. Бот будет способен выполнять ряд функций: 
1. Поиск в словаре по введенным китайским иероглифам или по русскому слову для перевода с примерами употребления. 
2. Поиск слов по пиньиню (китайской транскрипции). 
3. Создание карточек с иероглифами. 
4. Проверка знаний по созданным карточкам. 
5. Генерация теста в формате “вставьте недостающее слово/иероглиф”. 
Команды боту подаются через меню. После выбора команды бот отправляет информационное сообщение с уточнением запроса. Затем команда приходит к исполнению.

Пояснения к каждой команде по номерам:
1. После выбора команды, пользователю предлагается ввести китайское или русское слово. После этого бот ищет в словарях введенное слово и выводит пользователю перевод со всеми значениями, чтение (пиньинь) и 2-3 примера употребления в предложениях.
2. Пиньинь – это китайская система записи звуков латиницей и произношения иероглифов. В китайском языке используется система из 5 тональностей. Из-за этого многие слова записываются одинаково, но с разным знаком над акцентной буквой. После выбора команды пользователю предлагается ввести пиньинь без тональностей на английской клавиатуре. Бот ищет в словарях наиболее популярные слова с таким чтением и выводит результат в формате иероглифов, пиньиня (с тональностями) и перевода. Если слогов, введенных пользователем, не существует, то бот выводит соответствующее сообщение.
3. С помощью этой команды пользователь может ввести иероглиф (или китайское слово), и бот создаст с ним карточку, содержащую иероглиф, чтение и его значение. Созданные карточки можно просматривать по отдельной команде, а также удалять.
4. С помощью этой команды пользователь может пройти небольшой тест на знание слов из созданных им карточек. Бот пишет случайное китайское слово или иероглиф из карточек, а пользователь должен выбрать правильный перевод из 4 предложенных. Варианты выбираются также из карточек. После выбора бот отправляет сообщение об успехе, либо о неверном ответе. Если ответ неверный, то пользователю выводится полная карточка слова. После ответа тест можно продолжить, либо же прекратить.
5. При выборе данной команды, бот спрашивает, из скольких предложений составить тест (10, 15 или 20). Для использования команды, соответственно, у пользователя должно быть создано не менее 10 карточек. После выбора бот генерирует или берет из словарей соответствующее число предложений и посылает их пользователю по одному. В каждом предложении отсутствует иероглиф или слово из пользовательских карточек. Пользователю предлагается ввести недостающее слово. Бот запоминает ответ и присылает следующее предложение. Когда предложения заканчиваются, бот выводит результат и, по желанию пользователя, предложения, где он допустил ошибку, а также правильный ответ.

Описание меню:
1. Найти слово
При вводе команды запускается функции, описанные в пунктах 1 и 2 раздела “Описание функционала”. Бот выводит сообщение, описывающее ожидаемый ввод и вывод, ждет от пользователя ввода слова на русском, китайском языке или на латинице (пиньинь).
В панели команд отображается команда “Отменить”, которая прерывает команду и возвращает пользователя к главному меню.
2. Мои карточки
При вводе команды запускается функция, описанная в пункте 3 раздела “Описание функционала”. Бот выводит сообщение, описывающее механизм карточек. Далее пользователю отображается выбор действия: создать карточку или посмотреть карточки. 
При выборе первого варианта бот выводит сообщение, описывающее ожидаемый ввод и вывод. После успешного создания карточки выводится сообщение об успехе.
При выборе второго варианта бот выводит в сообщении пронумерованный список из китайских слов. Далее бот предлагает пользователю ввести номер желаемой к просмотру карточки. При корректном вводе отображается карточка слова с чтением и переводом, при некорректном — сообщение об ошибке и повторный запрос. 
В панели команд отображается команда “Отменить”, которая прерывает команду и возвращает пользователя к главному меню.
3.  Тест по карточкам
При вводе команды запускается функция, описанная в пункте 4 раздела “Описание функционала”. После сообщения с описанием функции бот предлагает выбор: начать или нет (через кнопки начать/отменить). Далее начинается тест, описанный выше, где варианты ответа генерируются в виде кнопок. Если ответ верный — выводится сообщение об успехе, если нет — отображается карточка слова, подобно пункту 2. После отображаются кнопки “продолжить тест” и “закончить”.
В панели команд отображается команда “Отменить”, которая прерывает команду и возвращает пользователя к главному меню.
4. Подбери слово
При вводе команды запускается функция, описанная в пункте 5 раздела “Описание функционала”. После сообщения с описанием функции бот предлагает выбор: начать или нет (через кнопки начать/отменить). Выбор количества вопросов осуществляется через сообщение с вопросом и кнопки “10”, “15” и “20”. После вывода предложения с пропущенным словом пользователь сам вводит недостающее. 
По окончании теста бот выводит результаты, отображающие в каких предложениях (по номерам) была допущена ошибка. Пользователь делает выбор с помощью кнопок: “Показать ошибки” или “Завершить”. При выборе первого поочередно отображаются предложения, в которых пользователь допустил ошибки, вариант пользователя и правильный ответ. При выборе второго варианта пользователь возвращается в главное меню.
В панели команд отображается команда “Отменить”, которая прерывает команду и возвращает пользователя к главному меню.

**Важно про “доступен только по ссылке”**
На GitHub это обычно `Private` репозиторий.  
Нюанс: приватный репозиторий **не откроется у другого человека просто по ссылке**, если вы не дали ему доступ (Invite collaborator).

Ниже пошаговый гайд через терминал VSCode.

## 1) Создать приватный репозиторий на GitHub
1. Откройте GitHub → `New repository`.
2. `Repository name`: например `chinabot`.
3. Выберите `Private`.
4. `Add README` лучше **не ставить** (чтобы проще запушить существующий проект).
5. Нажмите `Create repository`.

## 2) Подготовить проект в VSCode (терминал)
Откройте папку проекта в VSCode и выполните:

```bash
git --version
git init
git config user.name "Ваше Имя"
git config user.email "ваш_email@example.com"
```

Если `.gitignore` уже есть — отлично. Проверьте, что `.env` в нем есть.

## 3) Первый коммит
```bash
git add .
git status
git commit -m "Initial commit: Telegram Chinese learning bot"
```

Если случайно добавился `.env`:
```bash
git rm --cached .env
git commit -m "Remove .env from tracking"
```

## 4) Привязать удаленный репозиторий и отправить код
Скопируйте URL репозитория (например `https://github.com/USERNAME/chinabot.git`) и выполните:

```bash
git branch -M main
git remote add origin https://github.com/USERNAME/chinabot.git
git push -u origin main
```

## 5) Авторизация при push
GitHub попросит логин/пароль.  
Пароль = **не обычный**, а `Personal Access Token (PAT)`.

Если PAT еще нет:
1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic) → Generate.
2. Дайте минимум `repo`.
3. Скопируйте токен и используйте его как пароль в терминале.

## 6) Ссылка в отчет
Вставляете ссылку вида:
`https://github.com/USERNAME/chinabot`

## 7) Если преподаватель должен открыть приватный репозиторий
1. Репозиторий → `Settings` → `Collaborators` → `Add people`.
2. Добавьте GitHub-логин преподавателя.

Если хотите, дам такой же гайд в виде “копипаст-блока” только с командами под ваш репозиторий (с вашим `USERNAME/REPO`).
